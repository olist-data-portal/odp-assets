# コミットメッセージ提案

ステージングされている変更内容を分析して、過去のコミットメッセージを参考に適切なコミットメッセージを提案します。

## 実施内容

1. **ステージング済みファイルの確認**: `git status --staged`または`git diff --cached --name-only`でステージングされているファイルを確認
2. **変更内容の分析**: ステージングされている変更内容を`git diff --cached`で確認し、ファイルの種類と変更内容を分析
3. **コミットタイプの決定**: 以下のルールに基づいてタイプを決定
   - `docs:` - カールルール（`.cursor/rules/*.mdc`）やREADME、コメントの変更
   - `feat:` - 新規追加や破壊的変更
   - `fix:` - バグ修正
   - `refactor:` - リファクタリング（既存機能の改善、コード整理）
4. **メッセージ生成**: 過去のコミットメッセージを参考に、英語で1文のメッセージを生成
   - 最初は大文字で始める
   - 簡潔で明確な表現を使用
5. **和訳の追加**: 提案したメッセージの和訳も提供

## 過去のコミットメッセージ例

以下の形式を参考にしてください：
- `feat: Update terraform`
- `refactor: Update gitignore`
- `docs: Update rules and README`
- `feat: Add gitignore`
- `feat: Add terraform codes`
- `docs: Change cursor rules and README`

## 出力形式

以下の形式で提案してください。**提案文と和訳文は必ず別ブロックにすること**。提案文はそのままコピーしてコミットに使用できるようにする：

```
{type}: {Message}
```

```
和訳: {日本語訳}
```

**重要**: 
- 提案文（最初のブロック）は、そのまま`git commit -m`で使用できる形式で出力すること
- 和訳文（2番目のブロック）は参考情報として別ブロックで出力すること
- 2つのブロックは明確に分離すること

ステージングされている変更内容を分析して、適切なコミットメッセージを提案してください。
